# ============================================================
# SILENT DESYNC (app runs, no crash, but hooks don't work)
# ============================================================
# This happens when offset_encoder.py has been auto-updated to
# a new digest but the ENC_* constants in native-lib.cpp were
# never patched to match. --compute-digest will show DELTA=0
# because it only compares the binary to offset_encoder.py,
# NOT to the actual constants in the source code.
#
# THE LABEL KEY HAS THE SAME RISK: if PROLOGUE_DIGEST in offset_encoder.py
# was updated but label_key_encoder.py --compute wasn't re-run, the
# ENC_LABEL_OFFSET_* constants in native-lib.cpp will be stale and ON/OFF
# toggles will crash for legit users.
#
# HOW IT HAPPENS:
#   1. Someone runs --compute-digest (or it runs during a build)
#   2. offset_encoder.py gets auto-updated to the new digest
#   3. But patch_enc_constants.py + K1/K2 update + label_key_encoder.py
#      are NOT run
#   4. Next --compute-digest shows DELTA=0, hiding the problem
#
# SYMPTOMS:
#   - App launches fine (no crash)
#   - Prediction lines never appear / hooks silently fail
#   - With v2 cipher: decrypted strings are garbage even though the
#     bytes look fine in the binary (wrong K1/K2 = wrong output)
#   - ON/OFF toggles crash the app inside libart.so (label key stale)
#
# DIAGNOSIS:
#   1. Run: python offset_encoder.py
#      Compare the printed ENC_RVA_POWER_BAR_DRAGGED value against
#      the value in native-lib.cpp. If they differ, you have a
#      silent desync on the offset side.
#
#   2. Run: python label_key_encoder.py --verify
#      If it prints "FAILED — re-run --compute", you have a silent desync
#      on the label key side. Run --compute to fix.
#
# FIX:
#   The SAFEST method is ALWAYS the delta approach via patch_enc_constants.py.
#   It correctly handles BOTH single-layer (ENC_RVA_*) AND remote-protected
#   (ENC_REMOTE_*) constants because the delta only affects the g_rva_seed
#   component — the remote seed stays untouched.
#
#   *** CRITICAL: NEVER use encrypt_remote.py (legacy mode / no arguments) ***
#   *** to manually replace ENC_REMOTE_* constants!                        ***
#   *** Legacy mode uses fnv1a("turret95_magic") as the remote seed, but   ***
#   *** the runtime uses TARGET_REMOTE_SEED from the dual-pastebin system. ***
#   *** These are DIFFERENT values. The constants will LOOK valid but       ***
#   *** silently decode to wrong addresses — hooks install at garbage       ***
#   *** offsets and prediction lines never appear.                         ***
#
#   OPTION A — Delta patch (PREFERRED, always correct):
#     1. Find the DELTA between the old and new digest:
#        DELTA = OLD_DIGEST ^ NEW_DIGEST
#        (printed by --compute-digest, or compute manually)
#     2. Run: python patch_enc_constants.py <DELTA_HEX> native-lib.cpp
#        This correctly patches ALL ENC_* and ENC_REMOTE_* constants.
#     3. Run: python compute_k1.py <NEW_DIGEST>
#        Paste K1 + K2 fragments into decryptStr.
#     4. Run: python label_key_encoder.py --compute
#     5. Run normal build steps 1, 1.5, 2-5.
#
#   OPTION B — From scratch (if delta is unknown / state is uncertain):
#     1. Run: python offset_encoder.py
#        This prints the correct ENC_RVA_* values (single-layer ONLY).
#        Manually replace ONLY the single-layer ENC_RVA_* and ENC_BSS_*
#        constants in native-lib.cpp with these values.
#     2. For ENC_REMOTE_* constants: you CANNOT safely recompute these
#        from scratch without knowing TARGET_REMOTE_SEED (which lives
#        only in encrypt_remote.py and is NOT the legacy single-word seed).
#        Instead:
#          a) If you know the last working digest, compute
#             DELTA = LAST_WORKING_DIGEST ^ CURRENT_DIGEST and use
#             patch_enc_constants.py on just the ENC_REMOTE_* lines.
#          b) Or: keep a backup of the last known-good native-lib.cpp
#             and delta-patch from there.
#          c) Or: use encrypt_remote.py's "rotate" command (NOT legacy
#             mode) which uses TARGET_REMOTE_SEED correctly — but this
#             also rotates the pastebin words, so only use if you also
#             update both pastebins.
#     3. Run: python compute_k1.py <CURRENT_DIGEST>
#        Paste K1, K2_p1, K2_p2, K2_p3 lines into decryptStr.
#     4. Run: python label_key_encoder.py --compute
#     5. Run normal build steps 1, 1.5, 2-5.
#
# PREVENTION:
#   After EVERY --compute-digest that shows a non-zero DELTA,
#   IMMEDIATELY run patch_enc_constants.py + compute_k1.py +
#   label_key_encoder.py --compute before doing anything else.
#   Never leave offset_encoder.py updated without also updating
#   the source constants AND the label key offsets.
